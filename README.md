# Signaloid-Demo-WebPerformance-ProbabilisticLighthouseScoring

Probabilistic evaluation of web performance: capturing distributions from N Lighthouse audits to propagate uncertainty using the [Signaloid C0 platform](https://signaloid.io). Instead of a single deterministic score, this project captures the _distribution_ of your site's Lighthouse performance across hundreds of real audits, then propagates that uncertainty through the composite score calculation to answer: **how confident are we that our site scores ≥ 90?**

## Methodology

1. **Data Acquisition**:

   `scripts/run_audit.py` executes 1,000 headless Lighthouse audits against a target URL, saving each as a JSON report.

2. **Data Normalization**:

   `scripts/extract.py` parses the JSON reports, extracting `FCP`, `LCP`, `TBT`, `CLS`, and `Speed Index`, normalizes them to a 0–100 scale and writes per-metric sample files used as input to the C++ program.

3. **Probabilistic Computation**:

   `src/main.cpp` runs on the Signaloid C0 platform. Loads each metric's samples as an uncertainty distribution via `UxHwDoubleDistFromSamples`, computes the weighted composite score, and outputs the probability that the score falls below the "Good" threshold of 90.

4. **Visualization**:

   The Signaloid CLI generates Probability Density Functions (PDFs) for each metric and the final composite score.

## Metrics & weights

The following weights are based on the Lighthouse v10 performance scoring algorithm (see: https://developer.chrome.com/docs/lighthouse/performance/performance-scoring#lighthouse_10).

| Metric                         | Weight |
| ------------------------------ | ------ |
| First Contentful Paint (FCP)   | 10%    |
| Speed Index (SI)               | 10%    |
| Largest Contentful Paint (LCP) | 25%    |
| Total Blocking Time (TBT)      | 30%    |
| Cumulative Layout Shift (CLS)  | 25%    |

## Input

Each metric is collected as a series of per-audit scores normalised to the 0–100 scale. The composite Lighthouse performance score is computed as the weighted sum:

$$\text{score} = \sum_{i} w_i \cdot m_i = w_{\text{FCP}}\,\text{FCP} + w_{\text{SI}}\,\text{SI} + w_{\text{LCP}}\,\text{LCP} + w_{\text{TBT}}\,\text{TBT} + w_{\text{CLS}}\,\text{CLS}$$

where $m_i \in [0, 100]$ is the per-audit metric score and $\sum_i w_i = 1$.

On the Signaloid platform, each metric is represented not as a scalar but as a distribution fitted from the sample series. The weighted sum then propagates uncertainty through the formula, yielding a distributional composite score rather than a point estimate.

Input files are plain text, one per metric, located in `inputs/`:

```
<n>
<score_1>
<score_2>
...
<score_n>
```

Where `n` is the total number of Lighthouse runs and each `score_i` is the per-audit metric score in [0, 100].

## Getting started

### Prerequisites

Ensure the following tools and accounts are installed and configured:

- Node.js (v22.x or later) with npm or bun package manager

- Python 3.12 or later, with [uv](https://docs.astral.sh/uv/) for dependency management

- A Signaloid Cloud Developer Platform account. Sign up at [signaloid.io](https://signaloid.io)

- Signaloid CLI (`signaloid-cli`) for local execution. See the [installation guide](https://docs.signaloid.io/docs/api/signaloid-cli/installation/)

### Install Lighthouse globally

```bash
bun add -g lighthouse
```

### Install Python dependencies

```bash
uv sync
```

### Run audits

```bash
uv run python scripts/run_audit.py
```

This will run 1000 audits against the configured URL and save JSON reports to `reports/`. Edit `URL` and `TOTAL_RUNS` at the top of the script as needed.

> **Note:** Keep `MAX_WORKERS` low (2–4) or set it to 1 for valid IID samples. High parallelism causes Chrome instances to compete for resources, artificially inflating metric variance.

### Extract metric samples

```bash
uv run python scripts/extract.py
```

Reads all reports and writes `input/{fcp,lcp,tbt,cls,si}.txt`. Each file starts with the sample count, followed by one score (0–100) per line.

### Build & run

#### Using the API

Install dependencies and configure environment variables:

```bash
cd api
bun install
cp example.env .env
```

Populate `.env` with the following:

| Variable             | Description                                                                 |
| -------------------- | --------------------------------------------------------------------------- |
| `SIGNALOID_API_KEY`  | API key from [signaloid.io/settings/api](https://signaloid.io/settings/api) |
| `GITHUB_REPO_URL`    | HTTPS URL of the connected GitHub repository                                |
| `GITHUB_REPO_BRANCH` | Target branch (default: `main`)                                             |
| `SIGNALOID_CORE_ID`  | Signaloid core ID (default: C0-S+ uncertainty-tracking core)                |
| `SIGNALOID_URL`      | API base URL (default: `https://api.signaloid.io`)                          |

Run the full pipeline, it connects the repository, submits a build, waits for completion, executes a task against the uploaded input files, and retrieves the distributional output:

```bash
bun run src/index.ts
```

The pipeline executes the following steps in sequence:

1. Connects the GitHub repository to your Signaloid account (or reuses an existing connection).
2. Submits a repository build targeting the `src/` directory.
3. Polls build status until a terminal state is reached; surfaces the compiler log on failure.
4. Submits a task with the configured data source mounted at `sd0/`.
5. Polls task status until completion and retrieves stdout.
6. Extracts and summarises all Ux distribution strings from the output.

#### Local (signaloid-cli)

Authenticate with your API key:

```bash
signaloid-cli auth login --api-key $SIGNALOID_API_KEY
```

See the [authentication guide](https://docs.signaloid.io/docs/api/signaloid-cli/authentication/) for details.

Upload input files to Signaloid Cloud Storage:

```bash
signaloid-cli files upload --path inputs/fcp.txt --from ./inputs/fcp.txt
signaloid-cli files upload --path inputs/lcp.txt --from ./inputs/lcp.txt
signaloid-cli files upload --path inputs/tbt.txt --from ./inputs/tbt.txt
signaloid-cli files upload --path inputs/cls.txt --from ./inputs/cls.txt
signaloid-cli files upload --path inputs/si.txt  --from ./inputs/si.txt
```

Submit a build from source:

```bash
signaloid-cli builds create:source --file src/main.cpp --lang C++ --args "--enable-uncertainty"
```

Poll build status or wait for completion:

```bash
signaloid-cli builds status --build-id <build-id>
signaloid-cli builds watch  --build-id <build-id>
```

Submit a task, mounting the correct data source via `params.json`:

```bash
signaloid-cli tasks create --build-id <build-id> --params-file params.json
```

> The data source in `params.json` mounts your Signaloid Cloud Storage at `sd0/`. Ensure `src/main.cpp` reads input files from `inputs/{fcp,lcp,tbt,cls,si}.txt`.

Poll task status or wait for completion:

```bash
signaloid-cli tasks status --task-id <task-id>
signaloid-cli tasks watch  --task-id <task-id>
```

Retrieve task output:

```bash
signaloid-cli tasks output --task-id <task-id>
```

Generate distribution plots from the Ux strings in the output:

```bash
signaloid-cli plot ux-string --ux-string "<ux-string>" --out distributions
```
